# Step 3: Hybrid TurboQuant 完成

## 任务
实现 SnapKV 驱逐 + TurboQuant 量化混合（乘数效应）

## 文件变更
- `hybrid_quant.py` (25,136 bytes) — HybridTurboQuant 完整实现
- `_test_hybrid.py` (11,158 bytes) — 6 项测试，全部通过

## 核心组件

### 1. AttentionTracker
- 基于 Q-K 内积近似注意力分数（无需完整 attention 矩阵）
- 支持观察窗口模式（只用末尾 W 个 token 的 Q）
- EMA 累积更新：`score = 0.8 * old + 0.2 * new`
- 最近 token 额外权重（位置越近越重要）

### 2. TokenEvictor
- 4 种驱逐策略：heavy_hitter / window / hybrid / uniform_sample
- SnapKV 核心：保留 Top-K 重要 token（按累积注意力分数）
- expand()：将驱逐后的 KV 展开回原始长度（evicted 位置填零）

### 3. HybridTurboQuant
- 工作流：驱逐 → TurboQuant 压缩 → 解压 → 展开
- 内存降低公式：`1 / (p × avg_bits/16)` 倍
- 乘数效应：驱逐 × 量化

## 关键发现

1. **kept-position 质量很重要**：evicted 位置填零拉低 full-sequence CosSim，无意义
   → 正确度量：只对比 kept 位置的重建质量

2. **V=2 bit 理论下限**：Val CosSim ~0.99 是极限（4 个质心）

3. **乘数效应验证**：
   - 纯驱逐：~5x（保留 20%）
   - 纯 TurboQuant：~2.7x（4+2 bit）
   - 混合：~13.3x ✅（5.0 × 2.7 ≈ 13.5，理论预期达成）
